PROJECT 02 · 跨国政策研判 · 双引擎架构 · MULTI-AGENT

policy-observer · 双引擎跨国政策研判系统

客户类型:政策敏感行业(医药 / 能源 / 跨境贸易 / 教育)· 状态:设计与原型完成,持续磨合中

双引擎跨国政策研判系统——
国产 LLM 解读国内政策语境,海外 LLM 理解地缘叙事,
综合层做交叉研判,NGT 作为深度研判插件。

01 · 问题

客户场景与真实痛点

政策敏感行业(医药、能源、跨境贸易、教育等)对政策动向高度依赖,一条新政可能直接决定一个业务方向能不能继续做。

但单一信息源和单一模型容易产生偏差——国产模型对国内政策语境理解深,但对国际地缘叙事相对弱;海外模型对国际视角理解深,但对中文政策的「语境密度」识别有限。

客户描述的真实痛点是:「我们订了好几个研究服务,每家都说自己的视角对,但我没法判断到底信谁。我想要的不是更多信息,是更可靠的对比与综合。」

02 · 设计

产品逻辑与技术架构

双引擎并行,综合层裁定。国内观察站用国产 LLM 处理国内政策原文,全球观察站用海外 LLM 处理国际媒体与政策分析,综合引擎做交叉研判产出最终结论。NGT-AI 作为深度研判插件,处理复杂决策场景。

技术架构 · 三节点 + 插件
  • Node A国内观察站 — 国产模型,负责中文政策原文解读与本土语境
  • Node B全球观察站 — 海外模型,负责国际叙事与跨国视角
  • Node C综合引擎 — 接收 A / B 的产出,做交叉研判
  • + NGT深度研判插件 — 对高复杂度决策启用 6 阶段多 Agent 协同
  • 解耦三个节点可独立调用,也可组合订阅
03 · 架构图

数据如何流动

NODE A国内观察站国产 LLM · 中文政策原文
NODE B全球观察站海外 LLM · 国际地缘叙事
NODE C综合引擎接收 A / B 产出 · 交叉研判
PLUGINNGT-AI 深度研判插件仅重大决策点启用 · 6 阶段多 Agent 协同

双引擎并行输出 → 综合层交叉裁定。三个节点可独立调用，也可组合订阅。

04 · 演示

真实研究样张 · 与我怎么验它

架构讲完,关键是它产出的东西能不能信。这里放一段真实运行录屏、两张关键产出截图,以及我专门用来拷问它的方法。

样张 · 棉花目标价格政策研判(节选)
信源:发改价格〔2026〕733号 · 公开政策原文(733 字)

核心研判:政策本质从「保量稳价」切换为「控水提质 + 精准补贴 + 直售通道」——18600 元/吨目标价与 510 万吨固定补贴产量锁定基本盘,通过「非优势区退出 + 地下水超采调减」实施结构性收缩。

  • 二阶效应补贴固定化 + 质量导向,加速产业集团化整合;下游为规避目的国合规,转向「优质棉内销 + 进口棉出口」双轨,推高内贸优质棉溢价。
  • 风险点固定补贴 + 目标价机制,易被目的国援引 WTO《SCM 协定》认定为「可诉补贴」——出口企业需提前准备补贴性质界定的法律意见。
我怎么验它 · 接地度盲评

我没有止步于「能产出报告」。我把同一份政策喂给不同模型做盲评,专门查接地度——它说的每个数字(目标价 18600、固定产量 510 万吨…)能不能在信源原文里逐条找到?有没有冒出原文没有的「具体数据」?凡是查不到、又说得很具体的,就是疑似编造。

「能不能在原文里找到」这一关,才是决策类 AI 可不可信的分水岭。对系统更深一层的质量自审,见下方「现状」。

05 · 取舍

哪些路没走,为什么

为什么是模块化订阅,而不是强制打包?

不同客户对政策研判的颗粒度需求差异极大——小型企业可能只需要「国内观察」日报,大型机构需要全套综合研判。强制打包会让小客户买不起、大客户觉得不够细。

解耦设计让产品的客单价区间从一开始就能拉开,客户预算决定订阅深度。

为什么把 NGT 设计成「插件」而不是核心流程?

早期方案是把 NGT 写死在工作流里,每次研判都跑 6 阶段流程。后来发现:大部分日常政策追踪不需要 6 阶段深度协商,只有「重大决策点」(比如要不要进入某市场)才需要。

把 NGT 设计成插件后,系统的响应速度和算力成本都显著优化,同时保留了「必要时启用深度研判」的能力。

关于「双引擎」是否冗余

有人会问「用一个强大的模型不就行了吗」。但实际测试中,同一份政策文本,国产模型和海外模型的解读会在关键判断上分叉——分叉本身就是信号。

双引擎不是冗余,是结构化的「压力测试」。

06 · 现状

部署状态与可见性

设计与原型完成(5 包架构 + 115 个测试),在与目标行业场景持续磨合中。具体客户与部署细节不公开,代码仓库见 GitHub。

比起「看起来能用」,我更在意它「什么时候不该被信任」——我对它做过一次投资人级的质量自审,主动揪出「检索不到相关证据却仍自信研判」的问题,并定下三条修正方向:检索失败即熔断、可信度评分纳入信源相关性、强制量化与证伪条件。对决策类 AI,这种「知道自己何时不可信」的能力,才是真正的护城河。
