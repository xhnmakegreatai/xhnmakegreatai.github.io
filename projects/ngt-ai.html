PROJECT 03 · 多 AGENT 协同 · 原创方法论 · 开源

NGT-AI · 多 Agent 协同决策系统

类型:开源项目 · 状态:已发布在 GitHub

把社会学的名义小组技术(Nominal Group Technique)
系统化应用到多 LLM 协同决策中——
通过 6 阶段结构化讨论 + 强制使用异构模型,
从机制上对抗单模型偏见。

01 · 问题

客户场景与真实痛点

复杂决策场景中,单一 LLM 容易出现「偏见固执」——给出看似合理但忽略了关键反方观点的答案。用户追问时,模型会用更精致的论证维护原观点,而不是真正反思。

这不是某个具体模型的缺陷,是单模型架构的结构性问题:一个模型只有一个「世界观」,再聪明也只是单一视角。

现有的「多 Agent 系统」大多数是同一模型扮演不同角色,这种「假多元」在面对真正复杂的决策时,会因为底层模型一致而产生群体偏见——4 个 GPT 角色不会比 1 个 GPT 角色多多少视角。

02 · 设计

产品逻辑与技术架构

借鉴社会学的名义小组技术(NGT,1968 年由 Andre Delbecq 提出),一种用于避免群体决策中「从众效应」和「权威效应」的结构化方法。把这个方法应用到多 LLM 协同上,用强制的「独立产出 → 互不可见 → 交叉评分 → 立场表态 → 综合裁定」流程,让多个 AI 形成真正的多元视角。

技术架构 · 6 阶段流程
  1. 独立观点产出4 个 Discussant 各自独立思考,不接收其他人的输出
  2. 互不可见的方案输出4 个 Discussant 各自给出完整方案,系统隔离不交互
  3. 交叉评分每个 Discussant 给其他三个评分,不能给自己评分
  4. 看到评分评分结果公开,每个 Discussant 看到自己被评的分数
  5. 立场表态(REVISED / DEFENDED)每个 Discussant 必须选择:修订观点 或 坚持原观点(并说明理由)
  6. Referee 综合一个独立的 Referee 模型基于完整过程做综合裁定

04 · 取舍

哪些路没走,为什么

为什么强制 4 个 Discussant 使用不同厂商的模型?

这是整个系统的关键设计——GPT / Gemini / DeepSeek / Qwen 四个不同厂商的模型,训练数据、价值观倾向、推理风格都不同,它们的「独立观点」才是真正的独立。

用 4 个 GPT 扮演不同角色,得到的还是 GPT 的视角。异构是多元的物理基础,不是修辞。

为什么「互不可见」阶段不允许 Agent 之间交流?

这是 NGT 原始方法论的核心——一旦让 Agent 之间互相讨论,先发言者的观点会锚定后发言者,多元视角会快速收敛为单一视角。

互不可见不是不沟通,是为了沟通的质量。

REVISED / DEFENDED 为什么要二选一,不允许中间态?

真实决策中,「我觉得有点对又有点不对」是噪音,不是观点。强制二选一让每个 Discussant 必须做出明确表态,这些明确表态才能被 Referee 用来综合。

模糊的中间态会让综合环节失去信号。

05 · 技术路线

从方法论到可被直连的智能体

从一个社会学方法出发,按规范工程做成能跑、有测试、还能被标准协议直连的系统。

  • 方法论把名义小组技术(NGT)迁移到多 LLM 协同——异构模型是「多元视角」的物理基础,不是修辞。
  • 设计6 阶段编排 + 强制异构厂商模型 + 独立裁判;Provider 抽象,新增模型只实现一个接口。
  • 开发Python asyncio 编排器(并行扇出)+ FastAPI 后端 + React 前端;Mock 模式零依赖、克隆即跑。
  • 测试覆盖 provider / parser / 状态机 / 编排的单测,克隆后一条命令即可跑通全流程。
  • 互操作 · A2A内核已暴露为 A2A(Agent2Agent)智能体:Agent Card(/.well-known/agent.json)+ JSON-RPC 端点(/a2a),别的系统用标准协议就能直连它做决策。
